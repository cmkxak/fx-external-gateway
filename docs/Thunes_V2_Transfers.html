Transfers

견적으로 환율·수수료를 고정하고 → 거래 생성 → 확정으로 자금을 움직입니다. 타입이 Object ↗ 인 필드는 클릭하면 객체 상세로 이동합니다.

Discovery · Payers

송금 전에 가능한 서비스(코리도)와 지급처(payer)를 조회한다. payer 의 transaction_types 가 금액·필수필드 규칙의 단일 출처다.

Services 조회

계정에 활성화된 서비스(지급 수단/코리도) 목록.

Query

페이지 번호(기본 1).

페이지당 수(기본 50, 최대 100).
country_iso_codeString선택3
국가 필터(ISO 3166-1 alpha-3).

(반환)Service[]

견적 → 거래 생성 → 확정.

견적 생성

지급처(payer)와 금액으로 환율·수수료를 고정한 견적을 만듭니다. 견적은 expiration_date 까지만 유효합니다.

POST/v2/money-transfer/quotations

Body

external_idString필수64
우리 측 거래키(멱등키).
payer_idInteger필수
지급처 ID. 64-bit(Long) 사용.
modeString필수
금액 지정 기준. 선택한 쪽 amount 만 채우고 반대쪽은 null.
SOURCE_AMOUNTDESTINATION_AMOUNT
transaction_typeString필수
거래 유형. 우리는 C2C 기준.
C2CC2BB2CB2B
sourceSource필수
보내는(출발) 금액 정보.
destinationDestination필수
받는(도착) 금액 정보.

(반환)Quotation

금액은 Decimal, 통화별 precision/increment 규칙 준수(예: IDR precision=0·increment=1).

거래 생성

유효한 견적으로 거래를 생성합니다. 생성 직후 상태는 CREATED(10000). ext-{external_id} 변형 동일.

Body — V2 docs 기준

beneficiaryBeneficiaryC2C·B2C 필수
수취인. 거래유형에 따라 필수.

송금 사업자(C2C 미사용).
receiving_businessObjectC2B·B2B 필수

purpose_of_remittanceString필수64
송금 목적값. 예 FAMILY_SUPPORT. (허용값은 payer 의 purpose_of_remittance_values_accepted 로 제한될 수 있음)
document_reference_numberStringB2B 필수64

거래 확정

요청 바디 없음. 확정 시 source 금액 hold + 지급망 디스패치, 상태 CONFIRMED(20000).

POST/v2/money-transfer/transactions/{id}/confirm
POST/v2/money-transfer/transactions/ext-{external_id}/confirm

Path

idInteger필수
거래 ID.

Response · 200

(반환)Transaction
status=20000. 이후 최종 상태는 콜백/폴링.
주요 에러 — 1007002 이미 확정 · 1007004 견적 만료 · 1007005 잔액 부족.

}

Payer

지급처 객체. 금액·필수필드 규칙은 transaction_types[유형] 안에 있음.

OBJECT

Fields

idInteger
지급처 ID (= 견적 payer_id).
nameString
지급처(은행/월렛) 이름.
precisionInteger
금액 소수 자릿수 (THB=2, IDR=0).
incrementDecimal
금액 최소 단위(배수만 허용).
currency / country_iso_codeString
지급 통화 / 국가.
serviceService
지급 수단 객체 {id, name}.
transaction_typesObject
거래유형(C2C·C2B·B2C·B2B) 키를 갖는 맵. 각 유형 안에 금액·필수필드 규칙이 들어있다 — 이 객체가 요구사항의 단일 출처.
minimum_transaction_amountDecimal
최소 거래 금액.
maximum_transaction_amountDecimal | null
최대 거래 금액(null = 무제한).
credit_party_identifiers_acceptedArray[Array]
허용 수취 식별자 조합. 예 [["bank_account_number"]].
required_sending_entity_fieldsArray[Array]
송금인 필수 필드 조합(택1).
required_receiving_entity_fieldsArray[Array]
수취인 필수 필드 조합(택1).
required_documentsArray[Array]
필요 증빙 문서 조합([[]] = 불필요).
